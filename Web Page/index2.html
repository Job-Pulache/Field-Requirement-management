01 · GATEWAY — Issue Detection

What was broken?

The current process revealed 4 critical failures when mapped. Each became a decision point that blocked operational efficiency.

Issues
Found?
YES — 4 paths detected
Each path represents a process failure
node_01

No Digital Record

Requests were made verbally or on handwritten forms — impossible to track status or history.

node_02

No Approval SLA

No defined maximum response time — causing delays that impacted field operations every campaign.

node_03

No Rejection Log

Rejection reasons were never documented — the same issues repeated every cycle without resolution.

node_04

No Logistics Visibility

Logistics only learned of approvals via physical notification — creating last-minute bottlenecks.

05 · PARALLEL GATEWAY — Improvements

From AS-IS to TO-BE

4 improvements running in parallel — all implemented in the redesigned process.

4 paths
parallel
ALL — improvements applied simultaneously
improvement_01

Digital Request Registration

Every request is now logged digitally from the start — full traceability and historical analysis enabled.

improvement_02

24-Hour Response SLA

Management now has a defined maximum response window — operational delays in the field reduced.

improvement_03

Rejection Reason Tracking

Rejection reasons are recorded and communicated back to the field — enabling real continuous improvement.

improvement_04

Automatic Logistics Notification

Upon approval, logistics receives an automatic message event — eliminating last-minute bottlenecks.
